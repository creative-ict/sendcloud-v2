Checkout configuration

This is the final payload from the checkout middleware towards the integration.

Checkout Configuration

Acts as the root checkout configuration object:

Representation

  • id - String, The unique identifier of the checkout configuration
  • version - Number, The current version of the checkout configuration
  • updated_at - DateTime, JSON string representation of the checkout updated timestamp in UTC & ISO 8601 format (YYYY-MM-DDThh:mm:ss.sssZ)
  • delivery_zones - Array of Delivery Zone

Delivery Zone

All the information regarding the delivery zone is handled in the specific object.

Representation

  • id - String, The unique ID of the Delivery Zone
  • location - Object, The configured Location for this Delivery Zone
  • delivery_methods - Array of Delivery Method

Location

Describes the locaztion of the specific delivery zone (checkout for the first phase supports only domestic delivery zones).

Representation

  • country - Object, the destination Country

Country

Describes the origin / destination country for the specific location.

Representation

  • iso_2 - String, ISO2 representation of the Country
  • name - String, full official name of the Country

Delivery Method

A delivery method describes the delivery capabilities of a checkout project and is mix of Shipping Product information and merchant configured details.

Representation

  • id - String, The unique ID of the Delivery Method
  • delivery_method_type - String, describes the type of this delivery method (for the first phase of the project this will always be nominated_day_delivery).
  • shipping_product - Object, the [Shipping Product] (#Shipping-Product) The user selected Shipping Product
  • external_title - String, the integrations end user facing naming of the Delivery Method (e.g. "PostNL insured")
  • internal_title - String, the internal naming of the specific Delivery Method (to be used in admin interfaces)
  • carrier - Object, the Carrier
  • show_carrier_information_in_checkout - Boolean, switch to hide or display carrier information in the checkout page
  • shipping_product - Object, the configured Shipping Product
  • nominated_day_processing_days - Object, the merchant configured Processing days
  • lead_time_hours - Number, the Delivery Method lead times in hours

Carrier

Describes the carrier for the selected Delivery Zone

Representation

  • name - String, the name of the carrier
  • code - String, a unique identifier of the carrier
  • logo_url - String, a url pointing to the carrier logo

Shipping Product

Describes a Sendcloud Shipping Product and the user selected functionalities of it.

Representation

  • code - String, a unique identifier of a Shipping Product
  • name - String, user friendly naming of the Shipping Product
  • selected_functionalities - Object, the user selected functionalities for the specific Shipping Product.
  • delivery_days - Object, describes the Delivery Days of the carrier for the specific Shipping Product

Selected Functionalities

Describes the user selected functionalities of the configured Shipping Product

Representation

This is a dynamic object, current representation is out of the scope as the plugin itself is responsible of handling the data.

Processing Days

The merchant selected processing days with their cut off times.

Representation

  • day - Object, the day of the week represented as a string for every processing day of the merchant.
  • enabled - Boolean, enables / disables the selected day as a processing day
  • cut_off_time_hours - Number, the merchant's cut of time hours for the specified day
  • cut_off_time_minutes - Number, the merchant's cut of time minutes for the specified day
